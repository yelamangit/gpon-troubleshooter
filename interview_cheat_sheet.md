# Шпаргалка для собеседования: Архитектура и Код los_terminator

Этот документ поможет тебе уверенно рассказать, как работает твой проект под капотом, какие технологии используются и как ты интегрировал стороннее API.

## 1. Как работает интеграция с API Саппорт-центра (СЦ)?

На собеседовании скажи так: 
> *"Я интегрировал наш скрипт с внутренней тикет-системой через REST API. Для этого я использовал стандартную библиотеку `requests` в Python. В проекте я работаю с двумя основными HTTP-методами: GET и POST."*

**Что такое GET и POST в нашем скрипте?**
*   **GET-запросы:** Используются для **получения** данных. Например, чтобы скачать список открытых заявок.
    *   *В коде:* `requests.get(url, headers=headers)`. Мы шлем запрос на `/ticket-list?is_only_open=1` и система отдает нам JSON со списком тикетов. Ничего в базе СЦ мы при этом не меняем, просто читаем.
*   **POST-запросы:** Используются для **отправки** или **изменения** данных. 
    *   *В коде:* `requests.post(url, json=payload, headers=headers)`. Когда бот берет заявку в работу (`change-status`), оставляет комментарий с диагностикой (`ticket-reply`) или переводит тикет на ВОЛС (`assign-user`), он отправляет POST-запрос с JSON-телом (`payload`), в котором указано, что именно нужно изменить.
*   **API Токен (Авторизация):** Чтобы СЦ понимал, что запросы шлет авторизованный бот, а не хакер, мы передаем секретный ключ в заголовках (Headers): `Authorization: Bearer <твой_токен>`. Этот токен хранится безопасно в файле `.env`.

## 2. Как скрипт общается с коммутаторами (OLT)?

На собеседовании скажи так:
> *"Для взаимодействия с сетевым оборудованием я использовал библиотеку `netmiko`. Она идеально подходит для автоматизации CLI-интерфейсов."*

*   **Как это работает:** Скрипт по SSH или Telnet заходит на коммутатор, передает логин и пароль.
*   **Выполнение команд:** Скрипт отправляет команды (например, `show gpon interface ...`) и считывает весь текст, который коммутатор выплевывает в консоль.
*   **Пагинация:** Очень важная деталь, о которой стоит упомянуть! Сетевое оборудование обычно режет длинный текст и просит нажать пробел (`--More--`). Чтобы автоматизация не зависла, я отправляю команду `terminal length 0` (или `page-control disable` для ZTE), которая отключает эту пагинацию, и мы забираем весь лог целиком.

## 3. Парсинг данных (Regular Expressions)

На собеседовании скажи так:
> *"Оборудование возвращает сырой текст (Raw String). Чтобы вытащить оттуда конкретные цифры (например, затухание -29.6 dBm), я написал парсеры на основе регулярных выражений (RegEx)."*

*   Скрипт ищет в тексте паттерны. Например, регулярка `RxPower(?:.*?)(-?\d+\.\d+)` находит строку "RxPower", пропускает лишние символы и вытаскивает отрицательное число с точкой (наш сигнал).

## 4. Как устроен рабочий цикл (Daemon / Loop)?

На собеседовании скажи так:
> *"Скрипт работает в фоновом режиме (daemon) с помощью бесконечного цикла `while True`. Он скачивает свежие тикеты, пропускает их через логику, сохраняет ID обработанных тикетов в `Set` (чтобы не дублировать работу) и засыпает на 60 секунд через `time.sleep(60)`, не перегружая ни API, ни процессоры серверов."*
